---
title: /v1/extract/contexts
description: Determine the clinical context of concept spans - whether they are negated, historical, about a family member, or uncertain - with auditable rule triggers.
---

<EndpointHeader method="POST" path="/v1/extract/contexts" />

A SNOMED code alone does not tell you whether the patient *has* the condition. "Nie toute fievre" (denies any fever) and "antecedent familial de cancer du sein" (family history of breast cancer) both mention a concept that is not an active patient problem. `extract/contexts` applies rule-based ConText analysis (EN + FR lexicons) to each span and returns five context axes, each with the exact trigger phrase that fired so the decision is auditable.

Pass the spans (and optional codes) you got from [`/v1/extract/concepts`](/docs/api/extract-concepts).

## Request body

<Parameters>
  <Param name="text" type="string" required>The full clinical text the spans came from. After trimming, must be at least 2 characters.</Param>
  <Param name="language" type='"en" | "fr" | "auto"' defaultValue="auto">Language hint selecting the lexicon. <code>auto</code> infers it.</Param>
  <Param name="concepts" type="object[]" required>At least one span to analyze. Each item is `{ span: string, code?: string | null }`.</Param>
</Parameters>

<Parameters title="Context shape">
  <Param name="code" type="string | null">The SCTID you passed for the span, echoed back (may be <code>null</code>).</Param>
  <Param name="span" type="string">The span that was analyzed.</Param>
  <Param name="context" type="object">Five axes, each `{ value, confidence, trigger }` or <code>null</code> when not applicable.</Param>
  <Param name="context_confidence" type="number">Aggregate confidence across the axes.</Param>
  <Param name="readable_note" type="string">A short human-readable summary of the resolved context.</Param>
</Parameters>

<Parameters title="Context axes">
  <Param name="assertion" type='"affirmed" | "negated" | "uncertain"'>Whether the concept is present, denied, or unclear.</Param>
  <Param name="subject" type='"patient" | "family" | "other"'>Who the concept is about.</Param>
  <Param name="temporality" type='"current" | "past" | "chronic"'>When it applies.</Param>
  <Param name="certainty" type='"confirmed" | "differential" | "suspected"'>How certain the clinician is.</Param>
  <Param name="role" type='"finding" | "reason_for_encounter" | "history"'>The clinical role of the concept in the note.</Param>
</Parameters>

```typescript
type Axis = { value: string; confidence: number; trigger: string | null } | null;
type Context = {
  code: string | null;
  span: string;
  context: {
    assertion: Axis;
    subject: Axis;
    temporality: Axis;
    certainty: Axis;
    role: Axis;
  };
  context_confidence: number;
  readable_note: string;
};

async function extractContexts(
  text: string,
  concepts: { span: string; code?: string | null }[],
  language = "auto",
): Promise<Context[]> {
  const res = await fetch("https://api.obscyro.com/v1/extract/contexts", {
    method: "POST",
    headers: {
      Authorization: `Bearer ${process.env.OBS_KEY}`,
      "Content-Type": "application/json",
    },
    body: JSON.stringify({ text, language, concepts }),
  });
  if (!res.ok) throw new Error(`obscyro: ${res.status}`);
  const { contexts } = (await res.json()) as { contexts: Context[] };
  return contexts;
}
```

## Tips

- Every fired axis includes a `trigger` phrase, so you can show clinicians *why* a concept was marked negated or historical.
- A negated or family-subject `assertion` usually means the concept should not be added to the active problem list, even if `extract/concepts` resolved it confidently.
- Run [`/v1/extract/concepts`](/docs/api/extract-concepts) first to get the spans, then pass them here in one call.
