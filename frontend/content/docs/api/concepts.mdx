<EndpointHeader method="GET" path="/v1/concepts/{code}" />

Returns metadata and the preferred display term for a single SNOMED concept. Resolution prefers the shortest active synonym; if none exists, the FSN is returned.

## Path parameters

<Parameters>
  <Param name="code" type="string" required>
    A numeric SCTID. Must match the regex <code>^\d+$</code>.
  </Param>
</Parameters>

## Response

<ResponseExample status={200} description="Concept found">

```json
{
  "code": "22298006",
  "active": true,
  "effectiveTime": 20020131,
  "definitionStatus": "defined",
  "preferredTerm": "Myocardial infarction"
}
```

</ResponseExample>

<Parameters title="Response body">
  <Param name="code" type="string">The SCTID you asked for, echoed back.</Param>
  <Param name="active" type="boolean">Whether the concept is still in the active SNOMED edition. Inactive concepts are returned but flagged.</Param>
  <Param name="effectiveTime" type="integer">SNOMED <code>effectiveTime</code> stamp, in <code>YYYYMMDD</code> form, of the most recent change.</Param>
  <Param name="definitionStatus" type="string">One of <code>primitive</code> or <code>defined</code>.</Param>
  <Param name="preferredTerm" type="string | null">Shortest active synonym, falling back to FSN. <code>null</code> if the concept has no active descriptions.</Param>
</Parameters>

## Validating a code

Use **GET** to check whether a SCTID exists and retrieve its metadata. A `404 CONCEPT_NOT_FOUND` means the code is absent from the loaded SNOMED edition. A lightweight **HEAD** existence check (no response body) is planned for a future release.

## Errors

<ErrorTable>
  <ErrorRow status={400} code="VALIDATION_ERROR">`code` is not a numeric SCTID.</ErrorRow>
  <ErrorRow status={401} code="INVALID_API_KEY">Missing or invalid bearer token.</ErrorRow>
  <ErrorRow status={404} code="CONCEPT_NOT_FOUND">No concept with this SCTID exists in the SNOMED edition.</ErrorRow>
  <ErrorRow status={429} code="RATE_LIMITED">Per-minute rate limit exceeded for this key.</ErrorRow>
</ErrorTable>
