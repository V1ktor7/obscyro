<EndpointHeader method="POST" path="/v1/normalize" />

Normalize is the workhorse endpoint. Given a clinical phrase, it returns up to `limit` SNOMED concept candidates ranked by match quality. The matcher runs three strategies in a single batched query and deduplicates by concept:

1. **Exact** — case-insensitive equality against active descriptions.
2. **FTS** — PostgreSQL `to_tsvector` / `ts_rank` full-text search.
3. **Trigram** — `pg_trgm` similarity for typos and abbreviations.

## Request body

<Parameters>
  <Param name="text" type="string" required>The clinical phrase to match. After trimming, must be at least 2 characters; otherwise the request returns <code>400 INVALID_INPUT</code>.</Param>
  <Param name="limit" type="integer" defaultValue="5">How many candidates to return, between <code>1</code> and <code>50</code>.</Param>
</Parameters>

<Parameters title="Match shape">
  <Param name="code" type="string">SCTID of the matched concept.</Param>
  <Param name="term" type="string">The actual SNOMED description that matched (synonym or FSN).</Param>
  <Param name="conceptName" type="string">The preferred display term for the concept (shortest active synonym, falling back to FSN).</Param>
  <Param name="confidence" type="number">A score in <code>[0, 1]</code>. For <code>exact</code> it is 1.0; for <code>fts</code> it is the normalized <code>ts_rank</code>; for <code>trigram</code> it is the <code>similarity()</code> score.</Param>
  <Param name="matchType" type='"exact" | "fts" | "trigram"'>Which strategy produced this candidate. Use this to decide your minimum acceptable confidence per use case.</Param>
</Parameters>

```typescript
type Match = {
  code: string;
  term: string;
  conceptName: string;
  confidence: number;
  matchType: "exact" | "fts" | "trigram";
};

async function normalize(text: string, limit = 5): Promise<Match[]> {
  const res = await fetch("https://api.obscyro.com/v1/normalize", {
    method: "POST",
    headers: {
      Authorization: `Bearer ${process.env.OBS_KEY}`,
      "Content-Type": "application/json",
    },
    body: JSON.stringify({ text, limit }),
  });
  if (!res.ok) throw new Error(`obscyro: ${res.status}`);
  const { matches } = (await res.json()) as { matches: Match[] };
  return matches;
}
```

## Tips

- For ingestion pipelines that handle thousands of phrases, use [`/v1/normalize-batch`](/docs/api/normalize-batch) — it counts as a single rate-limit hit.
- If you need the right concept among multiple plausible matches, pass the candidates through [`/v1/disambiguate`](/docs/api/disambiguate) with surrounding clinical context.
- Trigram matches with confidence below ~0.55 are usually noise. Pick a threshold that suits your false-positive tolerance.
